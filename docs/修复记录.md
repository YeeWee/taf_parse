## 2026-03-18: VRB 风向显示修复

### 问题描述

解析不含 VRB 的 TAF 报文后，主体风的风向错误显示为 VRB（可变风向）。

**示例报文**:
```
FTDL31 EDZO 132300
TAF EDDF 132300Z 1400/1506 19010KT 9999 BKN020 PROB40 TEMPO 1400/1410 RA BKN010 BECMG 1401/1404 26006KT BECMG 1408/1412 27011KT PROB30 TEMPO 1414/1416 SHRA BKN025TCU BECMG 1416/1419 26004KT CAVOK PROB30 TEMPO 1500/1506 BKN020=
```

**问题表现**:
- 报文中风向从 `190°` 变为 `260°`，但解析后主体风显示 `variable=True`
- 时间线表格中，BECMG 期间（01:00-04:00）风向错误显示为 `VRB/5m/s`
- 航空用户无法获取准确的风向信息

**时间线对比**:
| 时间 | 主体风向 | 修复前显示 | 修复后显示 |
|------|----------|------------|------------|
| 14:00 | `190°` | 190°/5m/s | 190°/5m/s |
| 01:00-03:00 (BECMG 期间) | `190°` → `260°` | VRB/5m/s **← 错误** | 190°/5m/s **✓** |
| 04:00 (BECMG 完成) | `260°` | 260°/3m/s | 260°/3m/s |

### 根本原因

`src/parser.py` 的 `_get_worse_weather()` 函数（第 1199-1201 行）存在错误逻辑：

```python
# 风向：如果变化是 VRB（可变风向），优先使用 VRB
if change.wind.variable:
    worst.wind.variable = True
    worst.wind.direction = None  # VRB 时清空具体风向
elif change.wind.direction and worst.wind.direction:
    if change.wind.direction != worst.wind.direction:
        worst.wind.variable = True  # ← 错误：风向不同就设为 VRB
elif change.wind.direction and worst.wind.direction is None:
    worst.wind.direction = change.wind.direction
```

**问题分析**:
- BECMG（逐渐变化）期间，当变化前后风向不同时（如从 190° 变为 260°），代码错误地设置了 `variable=True`
- `_get_worse_weather()` 用于 BECMG 期间的较差值比较，风向不同只表示风向在变化过程中，不代表"可变风向"
- VRB（Variable）是 TAF 报文中的特定格式（如 `VRB03KT`），表示风向不定的状态，不应在比较两个固定风向时自动设置

### 修复方案

删除风向不同设置 VRB 的逻辑，VRB 标记只在原始报文包含 `VRB` 时才设置：

```python
# 修改前
elif change.wind.direction and worst.wind.direction:
    if change.wind.direction != worst.wind.direction:
        worst.wind.variable = True  # ← 删除此逻辑

# 修改后
elif change.wind.direction and worst.wind.direction is None:
    worst.wind.direction = change.wind.direction
```

**完整修改代码**:
```python
# 风向：如果变化是 VRB（可变风向），优先使用 VRB
if change.wind.variable:
    worst.wind.variable = True
    worst.wind.direction = None  # VRB 时清空具体风向
elif change.wind.direction and worst.wind.direction is None:
    worst.wind.direction = change.wind.direction
```

```
BECMG 期间     02:00 -> 190°/5m/s      ✓
BECMG 期间     03:00 -> 190°/5m/s      ✓
```

## 2026-03-18: TEMPO 降雪显示修复

### 问题描述

当主体天气为空但 TEMPO 有降雪时，时间线表格的状态列和天气列未正确显示降雪信息。

**示例报文**:
```
FTRS33 UUUJ 040500 AAB TAF AMD UUEE 040732Z 0407/0506 21005MPS 2500 -SN BKN010 TX00/0413Z TNM01/0409Z TEMPO 0407/0415 26008G13MPS 0700 +SHSN BKN005 BKN016CB BECMG 0415/0417 28005MPS 6000 NSW TEMPO 0418/0506 2100 -SN BKN006 =
```

**问题表现**:
- 18:00-05:00 期间，主体天气因 BECMG NSW 清除为空 `[]`
- TEMPO 有 `-SN`（小雪），但状态列显示"一般"，天气列显示"-"
- 航空用户无法看到 TEMPO 期间的降雪警告

**时间线对比**:
| 时间 | 主体天气 | TEMPO 天气 | 修复前状态 | 修复后状态 |
|------|----------|------------|------------|------------|
| 07:00-14:00 | `-SN` | `+SHSN` | 🌨️ 降雪 / 小雪 | 🌨️ 降雪 / 小雪 |
| 15:00-17:00 | `NSW` (空) | - | 🌤️ 一般 / - | 🌤️ 一般 / - |
| 18:00-05:00 | `[]` (空) | `-SN` | 🌤️ 一般 / -**← 错误** | 🌨️ 降雪 / 小雪 **✓** |

### 根本原因

`app.py` 时间线表格显示逻辑（第 278-310 行）存在两处问题：

1. **TEMPO 天气检查不完整**
   - 只检查了 `"TS"`（雷暴）和 `"RA"`（雨）
   - 未检查 `"SN"`/`"SHSN"`（降雪/阵雪）
   - 导致 TEMPO 降雪时状态仍为"一般"

2. **显示天气未使用 TEMPO 数据**
   - `weather_cn` 始终使用 `weather.weather`（主体天气）
   - 当主体天气为空但 TEMPO 有天气时，显示为"-"

**问题代码** (`app.py:278-310`):

## 2026-03-17: CB 云显示修复

### 问题描述

TAF 报文主体天气状态中的 `FEW035CB`（积雨云）格式，解析器正确识别了 CB 云类型，但显示时未展示云类型信息，导致航空用户无法看到重要的积雨云警告。

**示例报文**:
```
TAF VIDP 150500Z 1506/1612 07008KT 5000 HZ -RA SCT030 FEW035CB BKN090 TEMPO 1510/1514 25010G20KT 1500 -TSRA FEW035CB OVC080 BECMG 1518/1520 09006KT 3500 HZ BR FEW030 SCT080=
```

**问题表现**:
- 解析数据显示 `FEW035CB` 的 `type=CB`，但 UI 显示时只云量"少云"和高度"3500ft"
- TEMPO 明细表格同样未显示云类型
- 积雨云（CB）和浓积云（TCU）是重要的对流天气标识，对航空安全至关重要

### 根本原因

1. **`src/utils.py` 缺少云类型翻译函数**
   - 只有 `cloud_amount_to_cn()` 翻译云量（FEW/SCT/BKN 等）

2. **`app.py` 云显示逻辑未使用云类型**

### 修复方案

#### 1. 添加云类型翻译函数 (`src/utils.py`)

```python
def cloud_type_to_cn(cloud_type: Optional[str]) -> str:
    """
    云类型转中文

    Args:
        cloud_type: 云类型代码（CB/TCU 等）

    Returns:
        中文云类型描述

    if not cloud_type:
        return ""

        "CB": "积雨云",
        "TCU": "浓积云",
```

## 2026-03-17: 风切变预报支持

### 问题描述

无法解析北美 TAF 报文中的风切变预报 `WS020/21045KT` 格式。

**示例报文**:
```
TAF AMD KJFK 120308Z 1203/1306 21012KT 3SM -SHRA BR SCT060 BKN100 WS020/21045KT
FM120330 22012KT 2SM BR VCTS BKN005CB WS020/21045KT
FM120600 22012KT 2SM BR BKN005 WS020/21045KT
FM121100 31016G25KT P6SM -RA BKN025
PROB30 1218/1223 2SM -RASN BKN018
FM122300 31016G26KT P6SM BKN035=
```

**问题表现**:
- `WS020/21045KT` 格式的风切变预报不被识别
- 风切变信息丢失，无法提供给航空用户重要的低空风切变警告

**风切变格式说明**:
- 格式：`WShhh/wwwddffKT`
- `hhh`: 高度（百英尺），如 `020` = 2000 英尺
- `www`: 风向（度），如 `210` = 210 度
- `dd`: 风速（节），如 `45` = 45 节
- `ff`: 可选阵风

**示例**: `WS020/21045KT` = 2000 英尺高度，风向 210 度，风速 45 节

### 修复方案

#### 1. 添加风切变数据模型 (`src/models.py`)

```python
class WindShear(BaseModel):
    """风切变数据"""
    height: int = Field(..., description="风切变高度，单位英尺")
    direction: int = Field(..., description="风向，单位度")
    speed: int = Field(..., description="风速，单位 m/s")
    gust: Optional[int] = Field(None, description="阵风，单位 m/s")


class Wind(BaseModel):
    """风数据"""
    direction: Optional[int] = Field(None, description="风向，单位度")
    speed: Optional[int] = Field(None, description="风速，单位 m/s")
    gust: Optional[int] = Field(None, description="阵风，单位 m/s")
    variable: bool = Field(False, description="是否可变风向")
    wind_shear: Optional[WindShear] = Field(None, description="风切变预报")
```

#### 2. 添加风切变识别函数 (`src/parser.py`)

```python
def is_wind_shear_token(token: str) -> bool:
    """判断是否是风切变组"""
    # 格式：WShhh/wwwddffKT 或 WShhh/wwwddffMPS
    if token.startswith('WS'):
        remaining = token[2:]
        if '/' in remaining:
            parts = remaining.split('/')
            if len(parts) == 2:
                height_part = parts[0]
                wind_part = parts[1]
                # 高度应该是 3 位数字
                if height_part.isdigit() and len(height_part) == 3:
                    # 风速部分应该以 KT 或 MPS 结尾
                    if wind_part.endswith(('KT', 'MPS')):
                        return True
    return False
```

#### 3. 添加风切变解析函数 (`src/parser.py`)

```python
def parse_wind_shear(token: str) -> Optional[WindShear]:
    """
    解析风切变组

    格式：WShhh/wwwddffKT
    - hh: 高度（百英尺），如 020 = 2000 英尺
    - www: 风向（度），如 210 = 210 度
    - dd: 风速（节），如 45 = 45 节
    - ff: 可选阵风

    示例：WS020/21045KT = 2000 英尺高度，风向 210 度，风速 45 节
    """
    if not token.startswith('WS'):
        return None

    remaining = token[2:]
    if '/' not in remaining:
        return None

    parts = remaining.split('/')
    if len(parts) != 2:
        return None

    height_part = parts[0]
    wind_part = parts[1]

    # 解析高度（百英尺）
    if not height_part.isdigit() or len(height_part) != 3:
        return None

    height = int(height_part) * 100  # 转换为英尺

    # 解析单位
    unit = 'MPS'
    if wind_part.endswith('KT'):
        unit = 'KT'
        wind_part = wind_part[:-2]
    elif wind_part.endswith('MPS'):
        wind_part = wind_part[:-3]

    # 解析风向和风速
    # 格式：wwwdd(ff) - 3 位风向 + 2 位风速 + 可选 2 位阵风
    if len(wind_part) < 5:
        return None

    direction_str = wind_part[:3]
    speed_str = wind_part[3:5]
    gust_str = wind_part[5:] if len(wind_part) > 5 else None

    if not direction_str.isdigit() or not speed_str.isdigit():
        return None

    direction = int(direction_str)
    speed = int(speed_str)

    # 单位转换
    if unit == 'KT':
        speed = int(speed * 0.514444)

    gust = None
    if gust_str and gust_str.isdigit():
        gust = int(gust_str)
        if unit == 'KT':
            gust = int(gust * 0.514444)

    return WindShear(
        height=height,
        direction=direction,
        speed=speed,
        gust=gust
    )
```

```python
    """
```

## 2026-03-16: BECMG 期间取较差值支持

### 问题描述

BECMG（逐渐变化）变化组期间，应取变化前后要素的较差值，而不是只取变化后的值。

**示例报文**:
```
TAF EHAM 150506Z 1506/1612 27003KT 9999 FEW025
TEMPO 1506/1507 MIFG
BECMG 1509/1512 22012KT
BECMG 1519/1521 19017KT 7000 -RA BKN022
TEMPO 1521/1524 22018G28KT 4000 RA BKN014
BECMG 1523/1601 26014KT 9999 NSW FEW030
TEMPO 1600/1612 27017G27KT 6000 SHRA SHRAGS SCT025CB=
```

**问题表现**:
1. BECMG 1519/1521 期间（19:00-21:00），能见度从 9999 变为 7000，但解析器在 19:00-20:00 期间仍显示 9999
2. BECMG 期间云底高、风速等要素未取较差值
3. NSW（无重要天气）未正确清除天气现象

### 根本原因

1. **BECMG 期间逻辑不完整**
   - `get_weather_display_at_time()` 中，BECMG 只在 `query_time >= change.to_time` 时应用新天气
   - 在 `from_time <= query_time < to_time` 期间（变化进行中），未取前后较差值

2. **缺少较差值比较函数**
   - 没有 `_get_worse_weather()` 函数来比较两个天气状态

3. **NSW 处理不完整**
   - NSW 在 `_merge_weather()` 中未特殊处理，导致与降水同时存在

#### 1. 添加 `before_weather` 字段（`src/models.py`）

#### 2. 记录 BECMG 进行中的组（`src/parser.py:get_weather_display_at_time()`）

```python
def _get_worse_weather(base: WeatherState, change: WeatherState) -> WeatherState:
    """
    比较两个天气状态，返回较差值（用于 BECMG 期间）

    较差值定义：
    - 能见度：取较小值（更差）
    - 风速：取较大值（更差）
    - 阵风：取较大值（更差）
    - 云底高：取较低值（更差）
    - 天气现象：合并并按严重程度排序
    """
    worst = deepcopy(base)

    # 能见度：取较小值（更差）
    if change.visibility is not None:
        if worst.visibility is None or change.visibility < worst.visibility:
            worst.visibility = change.visibility

    # 风：取较大值（更差）
    if change.wind:
        if worst.wind is None:
            worst.wind = deepcopy(change.wind)
        else:
            if change.wind.speed and (worst.wind.speed is None or change.wind.speed > worst.wind.speed):
                worst.wind.speed = change.wind.speed
            if change.wind.gust and (worst.wind.gust is None or change.wind.gust > worst.wind.gust):
                worst.wind.gust = change.wind.gust

    # 云底高：取较低值（更差）
    # ... 找最低云底高逻辑

    # 天气现象：处理 NSW 和合并
    if 'NSW' in change.weather:
        worst.weather = []  # NSW 清除所有天气
    elif change.weather:
        all_weather = worst.weather + change.weather
        worst.weather = _merge_weather_phenomena(all_weather)

    return worst
```

```python
def parse_cloud(token: str) -> Optional[Cloud]:
    """解析云组"""
    cloud_amounts = ('SKC', 'FEW', 'SCT', 'BKN', 'OVC', 'VV')

    for amount in cloud_amounts:
        if token.startswith(amount):
            cloud = Cloud(amount=amount)
            remaining = token[len(amount):]

            # 解析云高（对于 VV，这是垂直能见度）
            if remaining and len(remaining) >= 3:
                # 支持 /// 格式（不明）
                if remaining[:3] == '///':
                    cloud.height = None  # 垂直能见度不明
                    remaining = remaining[3:]
                elif remaining[:3].isdigit():
                    cloud.height = int(remaining[:3]) * 100
                    remaining = remaining[3:]

            # 解析云类型
            if remaining in ('CB', 'TCU'):
                cloud.type = remaining

            return cloud

    return None
```
