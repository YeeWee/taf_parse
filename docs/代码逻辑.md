# TAF 解析器代码逻辑详解

## 1. 风要素的解码逻辑

### 关键字识别 (`is_wind_token`, 第 298-308 行)

- 以 `MPS` 或 `KT` 结尾
- 5 位以上数字开头
- `VRB` 开头（可变风向）

### 解析逻辑 (`parse_wind`, 第 311-358 行)

```
格式：dddffMPS 或 dddffGffMPS 或 VRBffMPS（公制）
      dddffKT 或 dddffGffKT（英制）

参数说明：
- ddd = 风向（度）
- ff = 风速
- Gff = 阵风值（G 分隔）
```

### 单位换算

| 单位 | 换算方法 |
|------|----------|
| KT（节） | × 0.514444 → m/s |
| MPS（米/秒） | 直接使用 |

### 风切变解析 (`parse_wind_shear`, 第 536-612 行)

```
格式：WShhh/wwwddffKT

参数说明：
- WS = 风切变标记
- hhh = 高度（百英尺），如 020 = 2000 英尺
- www = 风向（度）
- dd = 风速（节）
- ff = 可选阵风（G 分隔）

示例：WS020/21045KT = 2000 英尺高度，风向 210 度，风速 45 节
```

## 2. 能见度要素的解码逻辑

### 关键字识别 (`is_visibility_token`, 第 361-373 行)

- `CAVOK`：天气良好
- 米制：2-4 位纯数字（如 `6000`、`1500`）
- 英制：以 `SM` 结尾（如 `P6SM`、`5SM`、`1/2SM`）
- 带分数格式：整数 + 分数（如 `1` + `1/2SM`）

### 解析逻辑 (`parse_visibility`, 第 389-425 行)

| 格式 | 解析方法 | 换算 |
|------|----------|------|
| `CAVOK` | 返回 10000 米 | - |
| `6000`（纯数字） | 直接返回 | 米 |
| `P6SM` | 去掉 P，数值转换 | × 1609.34 |
| `1/2SM` | 分数计算 (1÷2) | × 1609.34 |
| `5SM` | 直接转换 | × 1609.34 |

### 北美特殊格式 (`parse_visibility_with_fraction`, 第 428-459 行)

```
格式：z x/ySM

示例：
"1" + "1/2SM" = 1 又 1/2 英里 = 1.5 英里 = 2414 米
"2" + "1/4SM" = 2 又 1/4 英里 = 2.25 英里 = 3620 米
```

## 3. 云及垂直能见度要素的解码逻辑

### 关键字识别 (`is_cloud_token`, 第 615-618 行)

云量代码：`SKC`、`FEW`、`SCT`、`BKN`、`OVC`、`VV`

### 解析逻辑 (`parse_cloud`, 第 621-646 行)

```
格式：AAAhht

参数说明：
- AAA = 云量（SKC/FEW/SCT/BKN/OVC/VV）
- hhh = 云高（百英尺），如 020 = 2000 英尺
- t = 云类型（CB/TCU），可选

示例：
BKN020 = 多云，云高 2000 英尺
SCT030CB = 疏云，云高 3000 英尺，积雨云
VV010 = 垂直能见度 1000 英尺
VV/// = 垂直能见度不明
```

### 单位换算

- 云高/垂直能见度：读取值 × 100 = 英尺

### 云量翻译 (`cloud_amount_to_cn`, 第 255-257 行)

| 代码 | 中文 | 说明 |
|------|------|------|
| SKC | 晴空 | Sky Clear |
| FEW | 少云 | Few (1-2/8) |
| SCT | 疏云 | Scattered (3-4/8) |
| BKN | 多云 | Broken (5-7/8) |
| OVC | 阴天 | Overcast (8/8) |
| VV | 垂直能见度 | Vertical Visibility |

### 云类型翻译 (`cloud_type_to_cn`, 第 260-278 行)

| 代码 | 中文 |
|------|------|
| CB | 积雨云 |
| TCU | 浓积云 |

## 4. 天气现象要素的解码逻辑

### 关键字识别 (`is_weather_token`, 第 462-513 行)

#### 前缀类型

| 类型 | 代码 | 说明 |
|------|------|------|
| 强度 | `+` | 强 |
| 强度 | `-` | 弱/小 |
| 特征 | `TS` | 雷暴 |
| 特征 | `SH` | 阵性 |
| 特征 | `FZ` | 冻结 |
| 特征 | `MI` | 浅 |
| 特征 | `BC` | 碎片 |
| 特征 | `PR` | 部分 |
| 特征 | `DR` | 低吹 |
| 特征 | `BL` | 高吹 |
| 位置 | `VC` | 附近 |
| 位置 | `RE` | 近来 |

#### 天气代码分类

```
降水类：
- DZ = 毛毛雨
- RA = 雨
- SN = 雪
- SG = 米雪
- IC = 冰晶
- PL = 冰粒
- GR = 冰雹
- GS = 小冰雹/雪粒

视程障碍类：
- FG = 雾
- BR = 轻雾
- FU = 烟
- VA = 火山灰
- DU = 浮尘
- SA = 沙
- HZ = 霾

其他类：
- PO = 尘/沙卷风
- SQ = 飑
- FC = 漏斗云
- SS = 沙暴
- DS = 尘暴
```

### 分级排序规则 (`_merge_weather_phenomena`, 第 996-1057 行)

| 等级 | 天气现象 | 评分 | 说明 |
|------|----------|------|------|
| 1（最严重） | TSGR | 95 | 雷暴 + 冰雹 |
| 1 | TSGS | 92 | 雷暴 + 小冰雹 |
| 1 | TSRA | 90 | 雷暴 + 雨 |
| 1 | TSSN | 85 | 雷暴 + 雪 |
| 2 | SS | 88 | 沙暴 |
| 2 | DS | 86 | 尘暴 |
| 3 | FZRA | 78 | 冻雨 |
| 3 | FZDZ | 76 | 冻毛毛雨 |
| 4 | GR | 75 | 冰雹 |
| 4 | GS | 70 | 小冰雹 |
| 5 | RA | 62 | 雨 |
| 5 | SN | 61 | 雪 |
| 6 | FG | 60 | 雾 |
| 6 | SHRA | 55 | 阵雨 |
| 7 | BR | 25 | 轻雾 |
| 7 | HZ | 32 | 霾 |

### 去重规则 (`_is_weather_superseded`, 第 1060-1118 行)

```
包含关系（保留更严重的）：
- TSRA 包含 RA、SHRA
- TSSN 包含 SN、SHSN
- FZRA 包含 RA
- FZDZ 包含 DZ
- FG 包含 MIFG、BCFG、PRFG
- BLSN 包含 DRSN
- +RA 包含 RA（强度包含）
- SHRA 包含 RA（阵性包含普通）
```

## 5. BECMG 的时间计算逻辑

### 时间解析 (`parse_change_group`, 第 724-754 行)

```
标准格式：BECMG DDHH/DDHH

示例：BECMG 1214/1218
表示在 12 日 14:00 至 12 日 18:00 期间逐渐变化
```

### FM/TL/AT 修饰符

| 修饰符 | 格式 | 说明 |
|--------|------|------|
| FM | BECMG FMddhhmm | 从指定时间开始变化 |
| TL | BECMG TLddhhmm | 到指定时间结束变化 |
| AT | BECMG ATddhhmm | 在指定时刻变化 |

### 替换规则 (`get_weather_display_at_time`, 第 845-851 行)

```
设查询时间为 t，BECMG 时间段为 [from_time, to_time]

t < from_time:
  → 使用原预报

from_time ≤ t < to_time:
  → 变化进行中，取较差值 (_get_worse_weather)
  → 能见度取小、风速取大、云底取低

t ≥ to_time:
  → 变化完成，使用新天气预报
```

### 较差值定义 (`_get_worse_weather`, 第 1154-1252 行)

| 要素 | 较差规则 |
|------|----------|
| 能见度 | 取较小值 |
| 风速 | 取较大值 |
| 阵风 | 取较大值 |
| 云底高 | 取较低值 |
| 天气现象 | 合并后取最严重 |

## 6. FM 的时间计算逻辑

### 时间解析 (`parse_change_group`, 第 704-708 行、第 758-762 行)

```
格式 1：FMddhhmm
示例：FM090600 = 从 9 日 06:00 开始

格式 2：FM DDHH/DDHH
示例：FM 0906/0912 = 从 9 日 06:00 开始
```

### 替换规则 (`get_weather_display_at_time`, 第 840-842 行)

```
设查询时间为 t，FM 时间为 fm_time

t < fm_time:
  → 保持原预报

t ≥ fm_time:
  → 完全替换为新天气预报 (_merge_weather)
  → 覆盖风、能见度、天气现象、云
```

### 特点

- FM 是**永久性变化**
- 完全覆盖之前的所有预报要素
- 有效期持续到下一个变化组或报文结束
- FM 之后的预报独立于 FM 之前

## 7. TEMPO、INTER 等短时变化的时间计算逻辑

### 时间解析 (`parse_change_group`, 第 664-723 行)

```
基本格式：TEMPO DDHH/DDHH
         INTER DDHH/DDHH

概率格式：PROB30 DDHH/DDHH
         PROB40 TEMPO DDHH/DDHH

组合格式：PROB30 INTER DDHH/DDHH
         INTER PROB30 DDHH/DDHH
```

### 支持类型（第 864 行）

| 类型 | 说明 |
|------|------|
| `TEMPO` | 临时变化（Temporary） |
| `INTER` | 间歇性变化（Intermittent） |
| `PROB` | 概率预报（PROB30、PROB40 等） |
| `PROB30 TEMPO` | 30% 概率的临时变化 |
| `PROB30 INTER` | 30% 概率的间歇变化 |
| `INTER PROB30` | 间歇性 30% 概率变化 |

### 时间计算（第 877-879 行）

```
设查询时间为 t，短时预报时间段为 [from_time, to_time]

from_time ≤ t < to_time:
  → 短时预报生效

注意：结束时间是开区间，to_time 时刻不再生效
```

### 替换规则（与主体部分的关系）

**不是替换，而是叠加**：

```
主体预报：持续有效
TEMPO/INTER: 在生效期间叠加到主体预报上

多个短时预报重叠时:
  → 取最坏情况 (_get_worst_tempo)
```

### 最坏情况聚合 (`_get_worst_tempo`, 第 919-993 行)

| 要素 | 聚合规则 |
|------|----------|
| 能见度 | 取最小值 |
| 云底高 | 取最低值（不分云量） |
| 风速 | 取最大值 |
| 阵风 | 取最大值 |
| 天气现象 | 合并后按严重程度排序，保留前 3 个 |

### 显示输出

- `main`: 主体天气预报（不含 TEMPO）
- `tempo`: TEMPO 最坏情况汇总
- `tempo_details`: 每个 TEMPO 组的明细
- `tempo_groups`: 所有生效的 TEMPO 组原始数据

## 8. 对报文全文需忽略部分的逻辑

### WMO 报头（第 50-51 行）

```
格式：XXXX YY ZZZZZZ

示例：FTUS31 KWBC 071740
      FTDL31 EDZO 132300

处理：直接跳过
```

### TAF 标记（第 38-47 行、第 59-61 行）

```
格式：
TAF ...
TAF AMD ...（修订报）
TAF COR ...（更正报）

处理：提取 TAF 后的内容，移除标记本身
```

### 报文结束标记（第 62-63 行）

```
标记：=

处理：替换为空格后忽略

示例：3000= → 3000
```

### 温度预报（第 133-135 行）

```
格式：
TXddhhZ - 最高温度（如 TX25/1206Z）
TNddhhZ - 最低温度（如 TN12/1203Z）

处理：暂时跳过不解析
```

### 备注部分（第 217-220 行）

```
标记：
RMK - 备注开始
NXT - 下一段
FCST BY - 预报员信息

处理：遇到这些标记后跳过后续内容
```

### 特殊清除标记

| 标记 | 含义 | 处理 |
|------|------|------|
| `NSW` | No Significant Weather | 清除天气现象列表 |
| `NCD` | No Cloud Detected | 清除云况列表 |
